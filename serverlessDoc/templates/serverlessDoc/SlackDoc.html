Serverless Function api endpoint: "To Send Messages To Slack Channel and workSpace"

Note:

All endpoint url are valid

Introduction

This Documentation contains a Serverless function written in Python. to send an Messages using your user_token or bot_token. Serverless Function component

Requirements

  • You need a Sack developer account. If you don't already have one,
  • Create a slack App to generate your USER_TOKEN AND BOT_TOKEN
  • Got to app
  • You need to add your slack developer USER_TOKEN or BOT_TOKEN to the connect to the Functions API.
  • (Httpie) is a command-line HTTP client. Its goal is to make CLI interaction with web services as human-friendly as possible. HTTPie is designed for testing, debugging, and generally interacting with APIs & HTTP servers.
  • Optional python Request libery
  • Curl client

    • Using the Functions Api

    Create Channel........

    Required argument case sensitive

  • Token
  • Name: e.g (sales)
  • Private: True or False (Default False)

    • To Create Channel using Httpie

    https https://faas-nyc1-2ef2e6cc.doserverless.co/api/v1/web/fn-15807169-84e8-4712-af3f-0667ee1f30ce/Create/Channel Token==Token Name=="sales"


    To Create Channel using curl

    curl -X POST -H 'Content-Type: application/json' {your-url} -d '"Token":"iuoueihuy7y3272980912","Name":"Sales",'

    Response

    { "Status": "Channel Created" }
